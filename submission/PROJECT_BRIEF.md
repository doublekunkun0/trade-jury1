# Trade Jury

## 一句话定位

Trade Jury 是一个基于 OKX OnchainOS 的链上交易陪审团。
它不是直接替用户下单，而是在交易前召集多个 AI Judge，从趋势、反身性、风险和执行质量四个维度进行听证，最后给出 `通过执行 / 条件通过 / 拒绝执行` 的裁决。

## 使用模型

- Claw 型号：OpenClaw 2026.3.2
- 大模型版本：openai-codex/gpt-5.3-codex

## 使用的 OnchainOS 能力

- Market：获取价格趋势、成交量、流动性、热点和风险线索
- Trade：获取最佳路由、报价、滑点和成交成本
- Wallet Context Ready：预留钱包上下文作为个性化风控输入

## 解决的问题

多数交易类 AI Agent 只会给“买 / 卖”结论，甚至直接推进执行。
真正高频、也最昂贵的错误，往往是以下几类：

- FOMO 追高
- 在流动性很差的时点重仓下单
- 只看方向，不看执行质量
- 风险偏好和交易标的不匹配

Trade Jury 把这些错误前置成一场“AI 听证会”，让系统有能力明确说出：

- 为什么支持这笔交易
- 为什么拒绝这笔交易
- 为什么只允许小仓位试单

## 核心亮点

1. 多 Agent 交叉质询
不是单一策略机器人，而是让不同角色分别发言，并保留冲突意见。

2. 可解释的执行闭环
不是只做看板或分析，而是从用户意图一路走到交易建议和最优路由。

3. 风控优先
Risk Judge 拥有一票否决权。系统不是“尽量帮用户成交”，而是“尽量避免不该做的成交”。

4. 非常适合演示
只需要一句自然语言输入，就能在 30 秒内展示完整流程：输入 -> 听证 -> 裁决 -> 执行建议。

## 评分点映射

### 结合度

核心闭环直接建立在 OnchainOS 的 `Market + Trade` 之上，不依赖难以演示的外围能力。

### 实用性

它解决的是普通交易者每天都在犯的错误，而不是一个只在少数场景有用的炫技工具。

### 创新性

把“多角色辩论 + 风险否决 + 执行裁决”组合成一个交易产品，在当前参赛方向里差异明显。

### 可复制性

Prompt 可复现、交互流程可复现、展示页和录屏逻辑也可复现。

## 最适合展示的三个场景

1. 追高 Meme：系统拒绝执行
2. 防守轮动：系统通过执行
3. 分批抄底：系统条件通过

这个组合有一个优势：
它能在很短时间内证明系统既会做，也会不做。
