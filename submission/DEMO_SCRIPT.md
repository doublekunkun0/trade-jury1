# Demo Script

目标：
用 30 秒把 `Trade Jury` 的差异化讲透，不讲废话。

## 版本一：30 秒

0s - 4s

打开演示页，停在 Hero 区域。
口播：
“我做的是 Trade Jury，一个不是直接替你下单，而是先开庭、再执行的链上 AI Agent。”

5s - 10s

把输入框里的默认案例保留为：
`Base 上把 2 ETH 换成 BRETT，目标 24 小时更高收益，最大回撤 4%。`

点击 `召集陪审团`。
口播：
“系统会先看 OnchainOS 的 Market 和 Trade 数据，再让 4 个 Judge 分别发言。”

11s - 17s

展示听证记录和最终裁决 `拒绝执行`。
口播：
“这里不是只会鼓励成交，而是明确告诉你为什么这笔交易不该做。” 

18s - 24s

点击 `防守轮动` 场景。
展示 `通过执行` 和建议仓位。
口播：
“当趋势变弱、路径干净、风险匹配时，它会允许执行，并给出仓位和路径建议。”

25s - 30s

下滑到评分区和 Prompt 区。
口播：
“Trade Jury 的核心创新是多 Agent 交叉质询 + 风险一票否决 + 可执行裁决，这让它比普通交易 bot 更像真正的 AI Agent。”

## 版本二：15 秒极短版

1. 开场一句：
“这是 Trade Jury，一个会先辩论、再执行的链上交易陪审团。”

2. 输入一个追高请求，展示 `拒绝执行`

3. 切到防守轮动，展示 `通过执行`

4. 收尾一句：
“它解决的不是怎么成交，而是哪些交易根本不该成交。”
